# Contributing to Goose TD

We love your input! We want to make contributing to Goose TD as easy and transparent as possible.

## Ways to Contribute

### 🐛 Reporting Bugs
- Use the [bug report template](https://github.com/sakarichanel-cmd/goose-tower-defense/issues/new?template=bug_report.md)
- Be as detailed as possible
- Include screenshots if applicable
- Test on multiple browsers if possible

### ✨ Suggesting Features
- Use the [feature request template](https://github.com/sakarichanel-cmd/goose-tower-defense/issues/new?template=feature_request.md)
- Check if the feature is already requested
- Explain your use case clearly

### 💬 Joining Discussions
- Share strategies and tips
- Ask questions about gameplay
- Suggest balance changes

### 📝 Improving Documentation
- Fix typos or unclear sections
- Add new guides or tutorials
- Improve existing documentation

## Development Guidelines

### Before You Start
1. Check [existing issues](https://github.com/sakarichanel-cmd/goose-tower-defense/issues) to avoid duplicates
2. Fork the repository
3. Create a feature branch: `git checkout -b feature/your-feature`

### Code Style
- Write clear, commented code
- Follow existing patterns
- Test thoroughly before submitting

### Commit Messages
- Use clear, descriptive titles
- Reference issues: `Fix #123`
- Keep messages concise but informative

### Pull Request Process
1. Update documentation if needed
2. Add tests for new features
3. Link to related issues
4. Describe your changes clearly

## Community Standards

- Be respectful and inclusive
- Provide constructive feedback
- Help others in discussions
- Report issues responsibly

## Questions?

- 💬 [Start a discussion](https://github.com/sakarichanel-cmd/goose-tower-defense/discussions)
- 🐛 [Check the FAQ](./docs/faq.md)
- ❓ [Open an issue](https://github.com/sakarichanel-cmd/goose-tower-defense/issues)

Thank you for contributing to Goose TD! 🦆
